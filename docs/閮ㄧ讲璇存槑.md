# 部署说明

## Vercel 部署

1. 将项目推送到 GitHub、GitLab 或 Bitbucket。
2. 在 Vercel 导入项目。
3. Build Command 使用 `pnpm build`。
4. 添加 `.env.example` 中的生产环境变量。

## 必填环境变量

```text
NEXT_PUBLIC_SITE_URL=https://你的域名
CMS_ADMIN_PASSWORD=高强度后台密码
SESSION_SECRET=至少32位随机字符串
```

## 推荐持久化配置

### 内容数据

创建 Upstash Redis 数据库并配置：

```text
UPSTASH_REDIS_REST_URL=
UPSTASH_REDIS_REST_TOKEN=
```

未配置时，后台保存只适合本地或具有持久磁盘的 Node.js 服务器。

### 图片与视频

在 Vercel 创建 Blob Store，并配置：

```text
BLOB_READ_WRITE_TOKEN=
```

配置后，后台上传素材会自动存储到 Vercel Blob。

## 域名与微信链接

将网站域名配置 HTTPS。若微信小程序 URL 使用了默认白名单之外的官方域名，通过以下变量增加：

```text
WECHAT_URL_ALLOWLIST=example.weixin.qq.com,example.com
```

多个域名使用英文逗号分隔。
